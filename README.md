# Executive Secretary (WhatsApp-first)

This project turns the baseline into a **single-master executive secretary system** designed for strategic communication over WhatsApp.

## What it does

- Enforces a single authority number (`MASTER_NUMBER`) with privileged command control.
- Maintains long-term memory (contacts, message history, commitments) in SQLite.
- Distinguishes contact hierarchy (master, close, professional, high-priority, unknown).
- Responds with natural, role-aware language tuned to sender tier and intent.
- Supports autonomous calendar triage behaviors (collects agenda/windows, prioritizes urgency).
- Exposes both generic JSON messaging endpoint and WhatsApp webhook endpoint.
- Includes controlled Chromium research module (`BrowserResearcher`) for verification workflows.

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e . pytest
uvicorn secretary.main:app --reload
```

## Environment

- `MASTER_NUMBER` (default: `+10000000000`)
- `MASTER_NAME` (default: `Master`)
- `DATABASE_PATH` (default: `secretary.db`)

## WhatsApp hookup

For Twilio WhatsApp sandbox or production sender:

- Configure webhook URL to `POST /webhooks/whatsapp`
- Twilio will provide `From` and `Body` form fields
- Replies are generated by the secretary engine with memory and hierarchy awareness

## Master commands (current)

- `tag <phone> <tier>`: Assign contact tier (`close`, `professional`, `high_priority`, `unknown`)
- `schedule <title> at <datetime>`: Create a commitment
